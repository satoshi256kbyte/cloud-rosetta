# Quickstart: 比較結果ページの本実装 検証ガイド

**Date**: 2026-07-05

## Prerequisites

- 003-frontend がデプロイ済み（Next.js + Amplify Hosting）
- DynamoDB に published 状態のテーマが存在する
- S3 に比較結果 JSON が存在する（004-ai-agent で生成したもの）
- `frontend/.env.local` に `STAGE=dev` が設定済み

## Validation Scenarios

### 1. テーマ一覧ページの表示確認

```bash
cd frontend
npm run dev
```

ブラウザで `http://localhost:3000` にアクセスする。

**Expected**:

- 公開済みテーマがカード形式で一覧表示される
- 各カードにテーマ名・比較軸数・対象プロバイダーが表示される
- テーマが 13 件以上の場合、ページネーションが表示される

### 2. 比較結果テーブルページの表示確認

ブラウザで `http://localhost:3000/comparisons/{themeId}/{axisId}` にアクセスする。

**Expected**:

- プロバイダーが列、属性が行のテーブルが表示される
- 各プロバイダーのサービス名・要約・参照元リンクが表示される
- 参照元リンクが新しいタブで開く
- 比較日時が表示される

### 3. テーマ別比較軸一覧ページ

ブラウザで `http://localhost:3000/comparisons/{themeId}` にアクセスする。

**Expected**:

- テーマに属する比較軸の一覧が表示される
- 比較軸が 1 つのみの場合、比較結果テーブルにリダイレクトされる

### 4. プロバイダーフィルタの動作確認

比較結果テーブルページでフィルタを操作する。

**Expected**:

- プロバイダーを選択/解除するとテーブル列が即座に更新される
- URL に `?providers=AWS,GCP` が反映される
- その URL をコピーして新しいタブで開くと同じフィルタ状態が再現される

### 5. モバイル表示の確認

ブラウザの DevTools で幅 375px に設定する。

**Expected**:

- 比較テーブルが横スクロール可能
- テーマカードが 1 列に縦並びになる
- フィルタ UI が操作可能

### 6. エラー・404 の確認

存在しない URL（例: `/comparisons/nonexistent/test`）にアクセスする。

**Expected**:

- 404 ページが表示される
- トップページへの導線が提供される

### 7. ISR 動作確認（本番環境）

Amplify にデプロイ後、ページにアクセスし、S3 のデータを更新して 1 時間後に
再アクセスする。

**Expected**:

- 更新後のデータが反映される（1 時間以内は古いデータのまま）
