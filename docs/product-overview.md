# cloud-rosetta プロダクト概要

## 概要

cloud-rosettaは、AWS・GCP・Azure・Akamai・Cloudflareなど、クラウドサービスを横断的に比較するためのサイトです。
サーバーレス系サービス、コンピューティング系サービス、CDNなど、テーマ別・比較軸別に整理された比較情報を提供します。

## 背景・目的

アーキテクチャ選定やプリセールスにおける技術選定を高速化することが、このプロダクトの目的です。
各社の公式サイトやサーバーワークスグループのエンジニアブログなどに散らばっている情報を、
比較に特化した形でまとめることで、意思決定にかかる時間を短縮します。

## 対象ユーザー（プロダクトの利用者）

以下の職種が、クラウドサービスの比較検討のために利用します。
本ドキュメントの読者（社内の開発者・AIエージェント）とは別の概念である点に注意してください。
読者については「本ドキュメントの位置づけ」を参照してください。

- 企画者
- 営業
- プロダクトオーナー
- 開発者

## 主要機能・ユースケース

- クラウドサービス同士の比較
  （例: サーバーレス系サービスの比較、コンピューティング系サービスの比較、CDNのAWS・GCP・Azure・Akamai・Cloudflare間の比較）
- 比較テーマの追加提案
- 比較軸（表の列に相当するもの）の追加提案
- AIエージェントによる比較の実行（公式サイト・エンジニアブログの参照、Web検索）
- AIエージェントが作成した比較結果を人間がレビューし、Pull Requestのマージによって公開する

比較テーマ・比較軸の追加提案をAIエージェントによる比較作業に着手させるかどうかは、人間が判断します。

## 全体ワークフロー

1. 社内メンバーがGitHub Issueで比較テーマ・比較軸を提案する
2. 人間がIssueをレビューする
3. レビューでOKであれば、人間がIssueに着手用のタグを付ける
4. タグ付けをトリガーにGitHub Actionsが起動し、AIエージェントが比較作業を実行する
5. AIエージェントは公式サイトやエンジニアブログをWeb検索で参照し、比較結果をMarkdownとしてPull Requestにまとめる
6. 人間がPull Requestをレビューする
7. Pull Requestのマージが公開の確定操作となり、内容がAmazon S3・DynamoDBへ同期される
8. 同期されたデータをもとに、Next.js（SSR）でページが生成・公開される

## 公開範囲・想定規模

- 初期フェーズは社内向けで、利用者は100人程度を想定しています。
- 将来的には外部公開も視野に入れています。

## 本ドキュメントの位置づけ

本ドキュメントは、社内の開発者およびAIエージェントを読者として想定しています。
Spec Kitを用いた仕様駆動開発を滞りなく進めるための前提情報を共有することを目的としています。

## 用語集

- Spec Kit: このプロダクト自体の開発で採用している、仕様駆動開発（Spec-Driven Development）のためのツール・フレームワーク。
- AgentCore: Amazon Bedrock AgentCoreのこと。比較作業を行うAIエージェントの実行基盤として利用する。
- AWS Knowledge MCP Server: AWSの公式ドキュメントを検索・参照するためのMCP（Model Context Protocol）サーバー。
  技術選定・設計時のAWS関連調査に利用する。
- cdk-nag: AWS CDKで定義したリソースに対して、セキュリティ・ベストプラクティスの観点からチェックを行うツール。
